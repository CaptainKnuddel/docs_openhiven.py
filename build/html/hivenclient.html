Client¶

Main objects used to connect to Hiven and wrap the Swarm and API

HivenClient¶

class HivenClient(args*, kwargs**)¶

Main Class used to interact with the Websocket and the Hiven API. Inherited by BotClient and UserClient

Parameters

  • token (str) – Token for the interaction with Hiven.

  • restart (bool) – If set to True will automatically restart the websocket once if an error occurred while running. Defaults to False

  • client_type (str) – Automatically set if UserClient or BotClient is used. Defaults to BotClient and will be automatically set if using BotClient or UserClient

  • event_handler (openhivenpy.events.EventHandler) – Event Handler Class that will be used to trigger events! Creates a new EventHandler on Default

  • heartbeat (int) – Heartbeat interval for sending a lifesignal to Hiven. Defaults to 30000 (in milliseconds)

  • log_ws_output (bool) – If set to True, the websocket will log automatically all incoming json data. Can provide even more insight to the Hiven Swarm. Defaults to False

  • ping_timout (int) – Seconds after the websocket will timeout after no successful pong response. More information on the websockets documentation. Defaults to 100

  • close_timeout (int) – Seconds after the websocket will timeout after the handshake didn’t complete successfully. Defaults to 20

  • ping_interval (int) –

    Interval for sending pings to the server. Defaults to None

    Note

    Changing the Ping Interval can cause the WebSocket connection to break, since currently when pinging Hiven, no response will be received and so the Connection would timeout itself!

  • event_loop (asyncio.AbstractEventLoop) – Event Loop that will be used for running the client and it’s methods

Raises

  • InvalidClientType – If a faulty Client Type was passed!

  • InvalidToken – If an empty, faulty or None Type Token was passed!

  • ImportError – If an required module is missing and not installed!
